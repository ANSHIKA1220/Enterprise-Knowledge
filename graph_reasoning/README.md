# Neo4j Graph Reasoning Module

## Overview

The **Graph Reasoning Module** is responsible for building and querying a Neo4j Knowledge Graph for the Enterprise Knowledge System.

This module integrates with the ingestion pipeline and converts enterprise documents, tickets, logs, and markdown files into structured graph relationships.

The graph enables:

* Root cause analysis
* Multi-hop reasoning
* Relationship discovery
* Enterprise dependency tracing
* Graph-based retrieval for AI agents

---

# System Architecture

```text
Enterprise Documents
        ↓
Anshika's Ingestion Pipeline
(parser → cleaner → chunker)
        ↓
Chunked Enterprise Data
        ↓
Graph Reasoning Module
(entity + relationship extraction)
        ↓
Neo4j Knowledge Graph
        ↓
Graph Retrieval
        ↓
LangGraph / LLM Agent
```

---

# Features

## Graph Construction

The module automatically:

* Reads ingestion pipeline chunks
* Extracts entities and relationships
* Creates graph nodes
* Creates graph edges
* Stores everything inside Neo4j

---

## Supported Node Types

| Node Type | Description                   |
| --------- | ----------------------------- |
| API       | Enterprise APIs and gateways  |
| Service   | Backend services and systems  |
| Database  | Databases and storage systems |
| Error     | Runtime and system errors     |
| Document  | Enterprise documents and logs |

---

## Supported Relationships

| Relationship | Meaning                       |
| ------------ | ----------------------------- |
| HAS_ERROR    | Service/API contains an error |
| CAUSED_BY    | Root cause relationship       |
| DEPENDS_ON   | Service dependency            |
| USES         | Technology or database usage  |
| MENTIONS     | Document references entity    |
| RELATED_TO   | General semantic relation     |
| PART_OF      | Entity hierarchy relation     |
| CAUSES       | Cause-effect relation         |

# File Descriptions

## graph_builder.py

Creates and inserts graph nodes and relationships into Neo4j.

Responsibilities:

* Neo4j connection
* Relationship insertion
* Graph creation
* Graph schema setup

---

## graph_retriever.py

Handles graph querying and retrieval.

Responsibilities:

* Entity extraction from user queries
* Multi-hop traversal
* Relationship retrieval
* Graph context generation

Main function:

```python
get_graph_answer_context(user_query)
```

---

## graph_from_ingestion.py

Integrates the graph module with the ingestion pipeline.

Responsibilities:

* Reads chunked enterprise data
* Extracts relationships
* Builds graph automatically from chunks

Uses:

```python
from ingestion_pipeline.ingestion.pipeline import ingest_directory
```

---

## test_graph.py

Testing file for graph retrieval.

Used to validate:

* Graph insertion
* Retrieval correctness
* Neo4j connectivity
